AI Player Props

The props tool is a separate app (FastAPI + Vite). The main site does not serve it on the same port, so you need both processes running on your machine, or set a hosted URL in production.

If you see “connection refused” on port 5179

Nothing is listening there yet. Start the backend, then the frontend (two terminals).

1. Backend (port 8101)

cd standalone-player-props/backend
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8101

2. Frontend (port 5179)

cd standalone-player-props/frontend
npm install
npm run dev

When Vite prints Local: http://localhost:5179/, use that exact URL in the browser. In dev mode the UI talks to the API through Vite’s /api proxy to 127.0.0.1:8101, so keep both uvicorn and npm run dev running. If you see connection refused on a link, either the dev server is not running or Flask has PLAYER_PROPS_URL set to a dead URL—unset it for local work or set it to a live props URL.

Advanced: set VITE_PROPS_API_BASE in frontend/.env to override the API base (skips the proxy when set).

Production

Set environment variable PLAYER_PROPS_URL on the Flask app to your deployed props URL. Visiting /player-props will then redirect there instead of showing this page.
